# Release Notes — February 25, 2026

## What's in this release

This release delivers two things:

1. **Individual Mode** — participants can now sign in, enroll in challenges, and have their progress saved automatically. Sprints unlock one at a time as they complete them.
2. **Admin improvements** — 14 usability fixes and enhancements to the admin panel and public challenge pages, based on the Feb 14 feedback round.

All changes are live on the existing production deployment. No separate instance was created — this is the same environment, updated in place.

---

## Feature 1: Individual Mode

### What it is

Previously, challenges were group/team-based only. Individual Mode allows a single participant to work through a challenge on their own, at their own pace, with their progress tracked and saved.

### How it works

- A participant visits a challenge page and signs in (or creates an account).
- They enroll in the challenge with one click.
- Sprints are presented in sequence — Sprint 1 is unlocked first. The participant completes assignments within that sprint.
- When all assignments in a sprint are marked complete, the next sprint unlocks automatically.
- If a sprint has a password (scratch card), the participant must enter it each browser session to access that sprint's content.
- Progress is saved on the server. The participant can close the browser, come back days later, sign in again, and pick up exactly where they left off.

### How to test it

1. Go to a challenge page (e.g., `/c/demochallengeEN`).
2. You should see a sign-in prompt if the challenge is set to individual or hybrid mode.
3. Sign in or create an account.
4. Click **Enroll** on the challenge page.
5. You'll be taken to the sprint view. Only Sprint 1 should be unlocked; the rest show as locked.
6. Open Sprint 1, complete all assignments within it (click the "Mark Complete" button on each).
7. After completing the last assignment in Sprint 1, you should see a celebration screen, and Sprint 2 should now be unlocked.
8. Sign out, close the browser, reopen it, sign back in — your progress should still be there.

## Feature 2: Admin UI Improvements

### 2.1 — Export Assignments from Challenge Detail

**What:** An "Export" button on the challenge detail page in admin. Downloads all assignments for that challenge as an Excel spreadsheet.

**How to test:** Go to Admin > Challenges > click any challenge > click the Export button in the header. An `.xlsx` file should download.

---

### 2.2 — Sprint Preview Links

**What:** Each sprint in the admin sprint list now has a preview link that opens the public view directly at that specific sprint (instead of the challenge start page).

**How to test:** Go to Admin > Challenges > click a challenge with sprints > in the Sprints section, click the preview/eye icon on any sprint. It should open the public page scrolled to or focused on that sprint. Each sprint also shows a copyable slug — click it to copy.

---

### 2.3 — Assignment Archive Filter

**What:** The assignments list on a challenge detail page now has an Active / Archived / All filter dropdown. Previously, archived assignments were mixed in with active ones.

**How to test:** Go to Admin > Challenges > click a challenge. The assignments section should show "Active" by default. Switch to "Archived" or "All" to see filtered results.

---

### 2.4 — Client Archiving

**What:** Clients can now be archived and restored. Archived clients are hidden from the default view but not deleted.

**How to test:** Go to Admin > Clients. Each client row has an "Archive" button. Click it — the client gets an "Archived" badge and disappears from the default Active view. Switch the filter to "Archived" to see it. Click "Restore" to bring it back.

---

### 2.5 — Bulk Quiz Question Import

**What:** A "Paste Questions" button in the quiz editor. You can paste multiple questions at once (one per line) instead of adding them one at a time.

**How to test:** Go to Admin > Assignments > edit or create a quiz-type assignment > open the quiz editor > click "Paste Questions." Paste several lines of text. Each line becomes a separate question. You can edit them individually after import.

---

### 2.6 — Quiz Questions Visible When Editing Assignments

**What:** When editing an existing quiz-type assignment, you can now see all quiz questions directly in the edit form (as a read-only preview) without opening the quiz editor.

**How to test:** Go to Admin > Assignments > edit a quiz assignment. Below the form fields, you should see a preview section listing all quiz questions with their type and options.

---

### 2.7 — Bulk-Add Assignments to a Challenge

**What:** When adding assignments from the library to a challenge, you can now select multiple assignments at once using checkboxes and link them all in one action.

**How to test:** Go to Admin > Challenges > click a challenge > click "Add Assignment" > in the library picker, check the boxes next to several assignments > click "Link Selected (N)" at the bottom.

---

### 2.8 — "Done" renamed to "Close" in Quiz Editor

**What:** The button at the bottom of the quiz editor now says "Close" instead of "Done" to make it clear that it closes the editor (saving happens automatically).

---

### 2.9 — Sprint Filter in Challenge Assignment List

**What:** A "Filter by Sprint" dropdown on the challenge assignments list. Lets you see only assignments from a specific sprint, or only unassigned assignments.

**How to test:** Go to Admin > Challenges > click a challenge that has sprints. A sprint dropdown should appear next to the archive filter. Select a sprint to filter.

---

### 2.10 — Vimeo Video Fix

**What:** Some Vimeo videos (especially private/unlisted ones with long URLs) were failing to load. The video embed system has been rebuilt to handle all Vimeo URL formats correctly, with a graceful fallback when a video can't load.

**How to test:** Open any assignment that has a Vimeo video. It should play normally. If a video can't be loaded, a message appears with a direct link to view it on Vimeo.

---

### 2.11 — Announcement Save Button Fix

**What:** The announcement save button was sometimes appearing disabled even when content had been entered. Fixed — it now correctly detects empty vs. non-empty content.

**How to test:** Go to Admin > create or edit an announcement. Type some text. The save button should be enabled. Clear all text — it should be disabled.

---

### 2.12 — Announcements Shown on Public Challenge Page

**What:** Announcements published for a challenge now appear on the public challenge home page, visible to all participants. Pinned announcements are highlighted.

**How to test:** Create an announcement for a challenge in Admin. Then visit the public challenge page (`/c/{slug}`). The announcement should appear below the challenge description. Pin it and reload — it should have an amber highlight with a pin icon.

---

### 2.13 — Quiz Content Simplified for Participants

**What:** When a participant opens a quiz-type assignment, they now see only the quiz questions — the instruction text, content body, and media/video sections are hidden to keep the focus on the quiz itself.

**How to test:** Open a quiz assignment from the public challenge view. You should see the title, cover image, and quiz questions only — no extra content blocks.

---

### 2.14 — Dropdown Fix in Assignment Library Dialog

**What:** When adding assignments to a challenge, dropdown menus inside the dialog were appearing behind the dialog. Fixed — dropdowns now correctly appear on top.

**How to test:** Go to Admin > Challenges > click a challenge > Add Assignment > in the picker dialog, click any dropdown (e.g., content type filter). It should appear on top of the dialog, not behind it.
