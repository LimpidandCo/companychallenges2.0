# Release Notes — February 25, 2026

## Overview

This release includes two major bodies of work:

1. **Individual Mode** — full build-out of authenticated user flows with server-backed progress persistence, sequential sprint unlocking, and enrollment management.
2. **Admin UI Improvements** — 14 tickets from the Feb 14 feedback round addressing usability, filtering, bulk operations, and bug fixes across the admin and public interfaces.

## Individual Mode

### What it does

Individual mode lets authenticated users enroll in challenges, track assignment completion server-side, and unlock sprints progressively. Progress persists across sessions — sign out, sign back in, and everything is where you left it.

### Database Changes

**Migration `010_individual_mode_sprints.sql`** (applied to remote)

- Added `sequential_sprints` BOOLEAN column to `challenges` table — controls whether sprints must be completed in order.
- Created `sprint_progress` table tracking per-user, per-sprint status (`locked`, `unlocked`, `in_progress`, `completed`) with timestamps and RLS policies.
- Indexes on `participant_id`, `sprint_id`, and `status`.

### Auth System

- `ClerkProvider` rewired to delegate to `MockAuthProvider` — `useUser()`, `useAuth()`, `SignInButton`, and `UserButton` all route through the mock auth system.
- Server-side `auth()` in `mock-server-auth.ts` now checks the `mock-auth-signed-in` cookie and returns `null` when not signed in (previously always returned a mock user).
- `DevAuthToolbar` positioned at top-right for testing sign-in/sign-out flows without interference.

### Server Actions

**`lib/actions/participants.ts`**
- `initializeSprintProgress()` — creates sprint progress entries for new enrollments.
- `getSprintProgressForChallenge()` — fetches all sprint progress for a user in a challenge.
- `completeSprintIfDone()` — auto-completes a sprint when all its assignments are done.
- `completeAssignment()` / `enrollInChallenge()` — updated to integrate sprint progress logic.

**`lib/actions/public.ts`**
- `getIndividualModeData()` — fetches user progress with lazy initialization for legacy enrollments missing `sprint_progress` rows.

### Public Pages

**Challenge Start Page** (`app/(public)/c/[slug]/start/`)
- Two-layer sprint locking: server-backed progress + session-only password unlocks via `sessionStorage`.
- Sprint landing pages with intro content before entering assignments.
- `SprintCard` component shows locked/unlocked/completed states.

**Assignment Page** (`app/(public)/a/[slug]/`)
- Sprint completion celebration on finishing a sprint's last assignment.
- Reads sprint unlock state from `sessionStorage`.

### Key Design Decisions

- Sprint passwords use `sessionStorage` (not `localStorage`) — intentional. Users must re-enter the scratch card password each browser session.
- Legacy enrollments get `sprint_progress` rows lazily on first access.
- `.maybeSingle()` used for next-sprint queries to handle graceful `null` returns.

## Admin UI Improvements (Feb 14 Tickets)

### 1. Export Assignments from Challenge Detail

**File:** `app/admin/challenges/[id]/page-client.tsx`

Export button added to challenge header. Downloads the challenge's assignments as an Excel file (`{slug}-assignments.xlsx`) using the existing `exportAssignments()` server action with `challengeId` filter.

### 2. Sprint Preview Links + Copyable Slugs

**File:** `components/admin/sprint-list.tsx`

- Preview link changed from `/{challengeSlug}/start` to `/{challengeSlug}/start?sprint={sprintId}` — opens directly into the specific sprint.
- Copyable slug button shows the sprint's slug in monospace with click-to-copy.

### 3. Assignment Active/Archived Filter

**File:** `app/admin/challenges/[id]/page-client.tsx`

Dropdown filter (Active / Archived / All) added to the assignments section header. Defaults to "Active". Filters by `assignment.archived_at` field. Empty state text adjusts when filters hide results vs. genuinely no assignments.

### 4. Client Archiving

**Migration:** `011_client_archive.sql` (pending remote apply)
- Adds `is_archived BOOLEAN DEFAULT false` to `clients` table.

**Files:** `lib/actions/clients.ts`, `components/admin/client-list.tsx`, `app/admin/clients/page-client.tsx`, `lib/types/database.ts`

- `archiveClient()` / `restoreClient()` server actions.
- Archive/Restore buttons (amber styling) on each client row.
- "Archived" badge on archived clients.
- Active/Archived/All filter dropdown on clients page.
- `Client` TypeScript interface updated with `is_archived`.

### 5. Bulk Quiz Question Import

**File:** `components/admin/micro-quiz-editor.tsx`

"Paste Questions" button in the quiz editor (both in list header and empty state). Opens a textarea accepting one question per line. All imported as `reflection` type. Live question count displayed. Editable after import.

### 6. Quiz Questions Visible in Assignment Edit

**File:** `components/admin/assignment-form.tsx`

When editing a quiz-type assignment, an `InlineQuizPreview` component renders showing all quiz questions with their type, options (for multiple choice), and a count footer. For new assignments, shows an info message.

### 7. Bulk Import Assignments into Challenge

**File:** `components/admin/assignment-picker.tsx`

- Checkboxes added to each assignment row in the library picker.
- `selectedIds` state (Set) tracks multi-selection.
- "Link Selected (N)" button appears in footer when items are checked.
- Calls `addAssignmentToChallenge` directly for each selected assignment, then reloads.
- Single-select flow (radio + Continue) remains unchanged for individual adds.

### 8. Rename "Done" to "Close" in Quiz Editor

**File:** `components/admin/micro-quiz-editor.tsx`

Footer button text changed from "Done" to "Close" to avoid confusion with saving.

### 9. Sprint Filter in Challenge Assignment List

**File:** `app/admin/challenges/[id]/page-client.tsx`

Sprint filter dropdown added alongside the archive filter. Options: "All Sprints", "No Sprint", and each sprint by name. Only appears when sprints exist.

### 10. Vimeo Video Error Fix

**File:** `app/(public)/a/[slug]/page-client.tsx`

- `getVimeoEmbedUrl()` rewritten to handle private video URL format (`vimeo.com/123456789/abc123def`) by extracting the path hash.
- Adds `dnt=1` privacy parameter.
- Logs warnings for unparseable URLs.
- New `VimeoEmbed` component wraps the iframe with error fallback UI — shows "Video could not be loaded" with a direct link to open on Vimeo.

### 11. Announcement Save Button Fix

**File:** `components/admin/announcement-form.tsx`

Disabled check changed from `!content.trim()` to `!content.replace(/<[^>]*>/g, '').trim()`. Strips HTML tags before checking emptiness — an empty TipTap editor returns `<p></p>` which was falsely passing the old check or confusing the enable/disable logic.

### 12. Announcements on Public Challenge Home Page

**Files:** `app/(public)/c/[slug]/page.tsx`, `app/(public)/c/[slug]/page-client.tsx`

- Server component now fetches `getPublicAnnouncements(challengeId)` in parallel with other data.
- Client component renders announcements after the description section.
- Pinned announcements highlighted with amber background and pin icon.
- Each shows title, publish date, and rich HTML content.

### 13. Quiz Assignment Content Hiding

**File:** `app/(public)/a/[slug]/page-client.tsx`

For quiz-type assignments (`content_type === 'quiz'`), instructions, content text, and media/video are hidden. Only the cover image, title, day label, quiz questions, and completion button remain visible.

### 14. Select Dropdown Z-Index in Dialog

**File:** `components/ui/select.tsx`

`SelectContent` z-index changed from `z-50` to `z-[300]`. Dialogs render at `z-[200]`. Since Radix Select uses a Portal, the dropdown now correctly renders above dialogs. Fixes the "Add to Challenge" dropdown being hidden behind its parent dialog in the assignment library.
